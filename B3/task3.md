# Понимание CI/CD

1. Push в ветку main.
2. Gitlab Runner запускает pipeline.
3. **Запуск тестов**:
   1. Статический анализ кода
   2. Юнит тесты
   3. Интеграционные тесты
   если один из шагов выволнинлся с ошибкой, то
   4. **Уведомление в Telegram** о провале с указанием номера шага
4. В случае успеха на шаге 3.4, **сборка докер образа**
5. **Пуш образа в Docker Hub**
6. Деплой на стенд
7. **Уведомление в Telegram** об успехе
